# 디지털 브로슈어 — 성균관대학교 학술정보관 AX 사례

## 개요

성균관대학교 학술정보관의 3대 AI 솔루션 시연에 중점을 둔 디지털 브로슈어.
HTML + CSS만으로 구현하며, iframe과 유튜브 임베딩을 활용하여 솔루션을 직접 체험하거나 시연 영상을 볼 수 있다.

## 산출물

동일한 카드 디자인을 두 가지 네비게이션으로 제공한다.

| 파일 | 네비게이션 | 용도 |
|------|-----------|------|
| `index-scroll.html` | 세로 스크롤 (원페이지) | 공유/열람용 |
| `index-slide.html` | 좌우 슬라이드 | 발표/시연용 |

## 비주얼 톤

- **다크 프리미엄**: 어두운 배경(`#0a1628`) + 블루 그라데이션(`#0f3460`)
- 액센트 컬러: `#4dabf7` (밝은 블루)
- 시연 iframe/영상이 어두운 배경 위에서 시각적으로 부각됨
- 폰트: 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)

## 카드 구조 (공통)

각 카드는 3요소로 구성:

```
┌─────────────────────────┐
│  [태그]                  │
│  헤드라인 (1줄 압축)       │
│                         │
│  ┌───────────────────┐  │
│  │                   │  │
│  │  비주얼 (60~70%)    │  │
│  │  iframe / 유튜브    │  │
│  │                   │  │
│  └───────────────────┘  │
│                         │
│  하이라이트 (핵심 정보)    │
│  [바로가기 링크] (선택)    │
└─────────────────────────┘
```

- **태그**: 솔루션 카테고리 (pill badge 형태)
- **헤드라인**: 기사 제목처럼 한 줄 압축. 독자의 시선을 멈추게 하는 역할
- **비주얼**: iframe 직접 시연 또는 유튜브 임베딩. 카드 면적의 60~70%
- **하이라이트**: 핵심 정보 1~2줄
- **링크 버튼**: 관리자 페이지 등 외부 URL 연결 (해당 카드만)

## 버전별 구현 상세

### A. 세로 스크롤 (`index-scroll.html`)

- 단일 HTML 파일, 외부 의존성 없음
- 모든 카드가 세로로 나열 (각 카드 100vh 또는 auto height)
- CSS `scroll-snap-type: y mandatory`로 카드 단위 스냅 스크롤
- 각 카드 사이 구분선 또는 미세한 간격

### B. 슬라이드 (`index-slide.html`)

- 단일 HTML 파일, 외부 의존성 없음
- 한 화면에 하나의 카드만 표시 (100vw × 100vh)
- 좌우 화살표 버튼 + 키보드 좌/우 화살표 지원
- 하단 도트 인디케이터 (현재 위치 표시)
- 슬라이드 전환: CSS transition (`transform: translateX`)
- JavaScript: 슬라이드 전환, 키보드 이벤트 처리 (최소한의 vanilla JS)

## 기술 제약

- HTML + CSS만 사용 (슬라이드 버전의 전환 로직에만 최소 vanilla JS 허용)
- 외부 라이브러리/프레임워크 없음
- 단일 HTML 파일 (CSS 인라인 또는 `<style>` 태그)
- 반응형: 데스크탑 우선, 모바일에서도 깨지지 않는 수준

## iframe 및 임베딩 주의사항

- iframe에 `sandbox` 속성 불필요 (직접 시연 목적)
- 유튜브 임베딩은 `youtube.com/embed/{id}` 형식 사용
- iframe에 `loading="lazy"` 적용하여 초기 로딩 최적화
- 추후 삽입 예정 카드는 placeholder UI 표시 (빈 영상 아이콘 + "준비 중" 텍스트)
