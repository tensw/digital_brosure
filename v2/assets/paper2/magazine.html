AI Core Summary

IETF I2NSF 프레임워크를 확장하여 Intent Fulfillment(의도 이행)과 Intent Assurance(의도 보증)를 모두 구현한 폐쇄루프 보안 제어 시스템 ICSC를 제안. Data Model Mapper를 통한 자동 정책 번역, ML 기반 실시간 위협 탐지, 자동 재구성을 포함하는 5단계 의도 기반 네트워크 보안 관리 프로세스를 설계하고 DDoS 시나리오로 성능을 검증하였다.

Key Findings
1
MTTD/MTTR 안정성: ICSC의 평균 탐지 시간(MTTD)과 평균 대응 시간(MTTR)은 클라이언트 수(5→25) 증가에도 거의 일정한 반면, 수동 운영은 지수적으로 증가. 25개 클라이언트 기준 수동 대비 약 5배 이상 빠른 대응.
2
Data Model Mapper 자동 매핑: Zhang-Shasha(ZSS) 트리 편집 거리 알고리즘으로 고수준 CFI YANG 모델과 저수준 NFI YANG 모델 간 자동 매핑 실현. PyangBind 기반 정책 자동 생성으로 완전 자동화된 Intent Fulfillment 달성.
3
Service Function Chaining(SFC) 자동 배포: Flow Classifier로 프로토콜별 트래픽을 분류하고, HTTP/HTTPS는 Firewall→Web Filter, 기타 프로토콜은 Firewall만 통과하도록 자동 구성. 다중 NSF 간 순차적 보안 적용 실현.
4
구독 기반 실시간 이상 탐지: Analyzer가 NSF와 구독 계약을 맺어 관련 데이터만 선별 수집. Decision Tree ML 알고리즘으로 DDoS 공격 패턴 실시간 탐지 및 자동 재구성(Rate Limiting, 경로 변경) 수행.
5
IETF 표준 기반 다중 벤더 호환: RFC 9315(IBN), RFC 8329(I2NSF) 등 IETF 표준 프레임워크와 YANG 데이터 모델을 채택하여 벤더 종속성 제거. IETF 해커톤을 통해 구현 검증 완료.

Methodology

System Design

I2NSF 프레임워크를 Monitoring Interface + Analytics Interface로 확장. Consumer-Facing, NSF-Facing, Registration 인터페이스와 Security Controller, Analyzer를 포함한 5단계 폐쇄루프 아키텍처 설계.

Implementation & Evaluation

OpenStack Ubuntu 20.04, Node.js + React(CFI), Python + MongoDB(번역), ConfD + NETCONF(적용), Suricata IDS/IPS. DDoS 시나리오에서 클라이언트 5–25개 증가 시 MTTD/MTTR 비교.

Practical Implications

Industry Application

고수준 보안 의도만 명시하면 정책 번역·배포·모니터링·적응이 자동 수행. 관리자의 전문지식 요구를 낮추고 대응 시간을 단축.

Network Operators

IETF 표준 YANG 모델 기반 벤더 종속 없이 다양한 NSF 통합 관리. SFC 기반 프로토콜별 자동 트래픽 라우팅으로 운영 복잡도 감소.

Standards Bodies

I2NSF에 Monitoring·Analytics Interface 추가 필요성을 실증. RFC 표준화 시 업계 전반의 IBN 보안 채택 가속화 가능.

AI Comprehensive Outlook

학술적 기여: Intent Fulfillment와 Intent Assurance를 IETF I2NSF 표준 위에서 동시에 구현한 최초의 체계적 시스템 설계. ZSS 알고리즘 기반 Data Model Mapper의 YANG 모델 자동 매핑은 기존 수동 매핑의 한계를 극복한 참신한 기여이다.

실용적 가치: 오픈소스 코드 + IETF 해커톤 검증으로 재현성 확보. MTTD/MTTR의 클라이언트 수 무관 안정성은 수동 운영 대비 확장성 우위를 입증한다. 산업 적용을 위해 정책 충돌 해결, DL 기반 위협 탐지, 5G/O-RAN 확장이 핵심 과제이다.
